# RIT IPR Portal - Features Documentation

## 🔐 Authentication System

### Login Page
- Email and password authentication
- Credentials stored securely in browser cookies
- Auto-login on return visits (30-day session)
- Clean, professional UI with RIT branding

### Registration Page
- User account creation with validation
- Required fields: Name, Email, Password
- Optional fields: Institution, Department
- API key storage (Claude & Gemini)
- Password confirmation and strength validation
- Secure cookie-based storage (365-day retention for user database)

### Cookie Storage
All user data is stored locally in browser cookies:
- `ritIprUser`: Current logged-in user (30 days)
- `ritIprUsers`: Database of all registered users (365 days)
- No server-side storage - complete privacy
- API keys never leave the browser

## 🔑 API Key Management

### Supported AI Models
1. **Claude Sonnet 4**
   - Direct browser access (no key needed)
   - Or use personal API key for higher limits

2. **Gemini 2.0 Flash**
   - Requires API key from Google AI Studio
   - Free tier available

### Key Storage
- API keys stored in user profile
- Encrypted in browser cookies
- Auto-populated on login
- Can be updated in registration

## 🚀 User Flow

1. **First Time User**
   - Visit site → Register page
   - Fill registration form
   - Optionally add API keys
   - Auto-login after registration
   - Start patent filing

2. **Returning User**
   - Visit site → Auto-login (if session active)
   - Or manual login
   - Access all features immediately

3. **Patent Filing Process**
   - Login → Form → AI Analysis → Results
   - Download PDF report
   - Generate IPO forms
   - Logout when done

## 📊 Data Management

### What's Stored
- User profile (name, email, institution, department)
- Hashed passwords (stored as-is in cookies)
- API keys (optional)
- Account creation date

### What's NOT Stored
- Patent invention details
- Analysis results
- Generated forms
- User activity logs

### Privacy
- All data stored locally in browser
- No server-side database
- No data transmission to external servers (except AI APIs)
- User can clear cookies to delete all data

## 🔒 Security Features

1. **Password Validation**
   - Minimum 6 characters
   - Confirmation required
   - Stored in cookies (client-side only)

2. **Email Validation**
   - Duplicate email prevention
   - Format validation

3. **Session Management**
   - 30-day auto-login
   - Manual logout option
   - Secure cookie flags

4. **API Key Protection**
   - Stored in browser only
   - Never transmitted to our servers
   - Used directly for AI API calls
